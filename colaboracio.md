# Explicació de Fork, Pull Request i Issue a GitHub

## Què és un Fork?

Un **fork** és una còpia d'un repositori GitHub que es crea a partir d'un projecte d'una altra persona o organització. Serveix per poder fer canvis de manera independent sense afectar l'original. És el primer pas per contribuir a projectes de codi obert o fer proves.

## Què és un Pull Request (PR)?

Un **Pull Request (PR)** és una sol·licitud per fusionar canvis que has fet al teu fork (o a una branca) amb el repositori original. És la manera formal de proposar millores, correccions o noves funcionalitats. Altres desenvolupadors poden revisar, comentar i aprovar aquests canvis abans d'integrar-los.

## Què és una Issue?

Una **issue** és un element de seguiment dins de GitHub que representa una tasca, un error o una proposta relacionada amb el projecte. Serveix per organitzar, discutir i prioritzar treballs pendents.

## Reflexió sobre el flux de treball a GitHub

El flux de treball de GitHub basat en forks, pull requests i issues facilita la col·laboració en equip perquè:

- Permet treballar de manera independent en forks sense afectar el codi original fins que s'aprovin els canvis.
- Facilita la revisió de codi i la discussió sobre canvis abans d'integrar-los.
- Centralitza la gestió de tasques i errors amb les issues, millorant l'organització.
- Dona transparència i historial clar dels canvis i debat, essencials en projectes col·laboratius.

Aquest model millora la qualitat del codi i fomenta la participació, especialment en projectes de codi obert.
